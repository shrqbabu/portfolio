Bookstore Data Analysis using SQL

Project Overview

This beginner-level project focuses on analyzing a bookstore database using SQL. The objective is to extract useful insights related to books, customers, sales, and inventory using structured queries. The database consists of three main tables: Books, Customers, and Orders. SQL queries are used to perform data filtering, aggregation, and analysis to simulate real-world business scenarios.

Tools & Technologies

  • SQL

Key Analysis Performed

  • Retrieved books based on genre and publication year
  • Identified customers from specific locations
  • Analyzed orders within a specific time period
  • Calculated total revenue and stock availability
  • Found most expensive and least stocked books
  • Identified repeat customers and high spenders
  • Analyzed sales distribution by genre and author
  • Calculated remaining inventory after fulfilling orders

Key Analysis Performed

The database includes:

  • Books (book_id, title, author, genre, price, stock)
  • Customers (customer_id, name, city, country)
  • Orders (order_id, customer_id, book_id, quantity, total_amount, order_date)

Relationships:

  • One customer can place multiple orders
  • One book can be part of multiple orders
